A Racket Tree

Recursive Tree

This page is just a link to my simple implementation of a recursive tree computer program. The recursive tree is defined as a V shape, or a V shape on top of the two ends of the previous V. This leads to some nice trees. It is obviously not meant to draw a natural looking tree. It was just fun to make this program and play with the parameters.

The conceptual parameters that alter the tree are roughly the following:

Depth
How deep the tree goes. The depth of a single V shape is 1.
Branch length
The length of the two branches at depth 1.
Branch separation angle
How much is the angle between the two branches at depth 1.
Branch length decay
The decay of the branch length defines the length of a branch at depth n (starting at depth 2). The length of a branch at depth n is the length of a branch at depth n-1 times the value of the branch length decay. The idea is that the decay value is between 0 and 1. The branches get smaller at every depth level.
Separation angle decay
Same idea as the branch length decay but on the separation angle. The idea is that the angle gets bigger of smaller at every depth.
Tree rotation
The tree rotation value is the angle of a V shape from depth 2 and deeper. The angle is relative to the angle of the V shape of depth n-1.
The correct implementation parameters can be found on the Github page.

Implementation

The program code is written in the Racket programming language. It can be found on my Github page. It is implemented as a recursive function (depth first). This may not be the most efficient way but I like it.

A tree is a list of lines. A line is a structure with two points. A point is a structure with two numbers representing 2D coordinates.

I included a simple GUI so you can play with the parameters :). The naming of the parameters may not be consistent. I made them up as I went writing the program, the Github README and this webpage.
